# 识货(Shihuo) 앱 가격 수집 자동화 — Claude Code 작업 지시서

> 이 문서를 Claude Code 첫 메시지에 인수인계서 v2와 함께 붙여넣으세요.

---

## 배경

识货(Shihuo)는 중국 가격 비교 플랫폼이다. 得物, 天猫, 淘宝, 京东 등 주요 중국 이커머스 가격을 한곳에서 비교할 수 있다. 우리는 이 앱에서 KREAM에 판매할 상품의 **중국 현지 원가(CNY)**를 자동 수집하려 한다.

현재 `china_price.py`가 `pyautogui` 기반으로 존재하지만, **좌표 기반 방식을 전면 폐기**하고 **스크린샷 + Claude API 비전** 방식으로 전환한다.

## 识货 앱 UI 구조 (Mac 데스크톱 앱)

앱은 **왼쪽: 홈 피드 / 오른쪽: 상세 패널** 2컬럼 구조이다.

### 화면별 플로우

**1) 홈 화면**
- 상단에 검색바 + 搜索(검색) 버튼
- 카테고리 탭: 鞋类, 服饰, 美妆, 箱包 등
- 추천 상품 피드 (이미지 + 상품명 + ¥가격)

**2) 검색창 클릭 시 (오른쪽 패널)**
- 历史搜索: 이전 검색어 태그 표시
- 热门发现: 트렌드 키워드
- 飙升榜: 급상승 순위

**3) 품번 검색 결과 (오른쪽 패널)**
- 검색창에 품번(예: id6016) 입력 → 搜索
- 결과: 상품 카드 리스트 (상품명 + 컬러별 가격 + 판매수)
- 상단 탭: 综合/新品/销量/价格
- 필터: 鞋面材质/送礼人群/鞋的尺码

**4) 상품 상세 (오른쪽 패널)**
- 상품 이미지 + 컬러/옵션 선택 (款式 1/N)
- **사이즈 바** — 좌우 스크롤, 각 사이즈별 가격 표시 (예: 38 ¥314, 39 ¥293)
  - 缺货 = 품절
- **가격 비교 카드 3개:**
  - 全网最低 (전체 최저): 가격 + 상점명
  - 官方最低 (공식 최저): 가격 + 상점명
  - 发货最快 (최빠 배송): 가격 + 상점명
- **识货价格走势** — 30/60/180일 가격 추세 그래프

**5) 스크롤 다운 — 상품 스펙**
- 品牌 랭킹, 판매량 정보
- **스펙 요약 바** (클릭 가능):
  - 货号(품번), 品牌(브랜드), 鞋面材质(소재), 鞋底材料(밑창), 闭合方式(체결), 流行元素(트렌드), 鞋帮高度(발목), 功能性(기능)
- 클릭하면 **产品参数 모달** 열림 — 전체 스펙 표시, 오른쪽 상단 X로 닫기

**6) 더 스크롤 — 사이즈 차트**
- 尺码对照表: EU/CHN/UK/US 사이즈 매핑
- 鞋款介绍: 상품 소개 이미지

**7) 查看全网底价 (전체 최저가 비교 화면)**
- 하단 빨간 버튼 "查看全网底价" 클릭으로 진입
- **상단 플랫폼별 가격 탭:** 全网 / 官方 / 天猫 / 得物 / 淘宝 / 京东
- **필터:** 低价优先 / 速度优先 / 识货补贴
- **판매처 리스트 (위→아래):** 각 항목마다:
  - 상점명 + 플랫폼 아이콘
  - 가격 + 배송 정보
  - 인증 표시 (品牌官方, 鉴别通过, 安心购, 假一赔三 등)
  - "前往购买" 버튼

## 수집 대상 데이터

각 상품에 대해 다음을 수집해야 한다:

```
필수 수집 항목:
- 货号 (품번/모델번호)
- 品牌 (브랜드)
- 상품명 (중국어)
- 컬러명
- 사이즈별 가격 (全网最低 기준)
- 플랫폼별 가격 (得物/天猫/官方/淘宝/京东)
- 수집 상점명 + 플랫폼명 (출처 필수 기록)
- 수집 시점 (타임스탬프)
- 품절 여부 (缺货)

선택 수집 항목:
- 产品参数 전체 스펙
- 가격 추세 (180일)
- 사이즈 차트 (EU/CHN/UK/US 매핑)
```

## 신뢰 가능 판매처 판별 규칙

**모든 상점의 가격을 수집하지 않는다.** 아래 조건에 맞는 채널만 수집한다.

### ✅ 수집 OK

| 조건 | 판별 방법 |
|------|-----------|
| **得物** | 상점명이 "得物"로 시작. 得物, 得物-品牌官方 등 모두 OK |
| **品牌官方** | 상점명 옆에 다른 색 글씨로 "品牌官方" 표시됨 |
| **官方旗舰店 / 旗舰店** | 상점명에 "官方旗舰店" 또는 "旗舰店" 포함 |
| **专营店** | 상점명에 "专营店" 포함 |

### ❌ 수집 제외

- 위 조건에 해당하지 않는 일반 상점
- "假一赔三"만 있고 플랫폼 인증이 없는 상점
- 识货 추천(为你推荐)은 참고만, 인증 표시로 최종 판단

## 자동화 수집 흐름 (china_price.py가 수행해야 할 순서)

모든 단계에서 **스크린샷 → Claude API 비전 → 판단 → 클릭/데이터 읽기** 패턴을 따른다.

```
1. 识货 앱 실행 확인 (또는 실행)
2. 스크린샷 → AI에게 "검색바 위치" 질문 → 클릭
3. 품번(모델번호) 입력 (pyautogui.typewrite 또는 pyperclip+paste)
4. 스크린샷 → AI에게 "搜索 버튼 위치" 질문 → 클릭
5. 스크린샷 → AI에게 "검색 결과 첫 번째 상품 위치" 질문 → 클릭
6. 스크린샷 → AI에게 "产品参数의 货号가 입력한 품번과 일치하는지" 질문 → 확인
7. 사이즈별 가격 수집:
   - 스크린샷 → AI에게 "보이는 사이즈 목록 + 가격 + 스크롤 필요 여부" 질문
   - 더 있으면 → 스크롤 → 다시 스크린샷 → 추가 사이즈 수집
   - 缺货이면 null 기록
8. 스크린샷 → AI에게 "查看全网底价 버튼 위치" 질문 → 클릭
9. 스크린샷 → AI에게 "판매처 리스트 + 신뢰 가능 여부" 질문 → 데이터 수집
10. 결과를 JSON/엑셀로 저장
11. 다음 상품으로 이동 (반복)
```

### 사이즈 수집 — AI가 알아서 판단

상품 카테고리에 따라 사이즈 개수가 다르다:
- **신발 (鞋类):** 10~20개+ → 스크롤 필요할 수 있음
- **의류 (服饰):** 3~6개 (XS~XXL 또는 FREE) → 보통 스크롤 불필요
- **가방/시계:** 1~3개 → 스크롤 불필요

AI가 스크린샷을 보고 "사이즈가 몇 개 보이는지", "오른쪽에 더 있는지"를 직접 판단하므로 하드코딩된 스크롤 로직이 필요 없다.

## ⚠️ 핵심 아키텍처: 좌표 기반 → AI 비전 방식으로 전환

### 기존 방식의 문제 (폐기)
- `pyautogui` 절대/상대 좌표 기반 → 캘리브레이션 필요, 앱 업데이트 시 깨짐
- 좌표를 외워서 클릭 → 사람처럼 화면을 "이해"하지 못함
- 캘리브레이션 자체가 사용자에게 부담 (마우스 올리고 Enter 반복)

### 새 방식: 스크린샷 + Claude API 비전

**원리:** 사람이 화면을 보고 판단하듯이, 매 단계마다 스크린샷을 찍고 Claude API에 보내서 "뭐가 보이는지", "어디를 클릭해야 하는지"를 판단한다.

**핵심 흐름:**
```
1. pyautogui.screenshot()으로 현재 화면 캡처
2. Claude API (claude-sonnet-4-20250514, vision)에 이미지 전송
3. 프롬프트: "이 识货 앱 화면에서 [검색바/사이즈38/查看全网底价 버튼]의 좌표를 알려줘"
4. Claude가 좌표 반환 → pyautogui.click(x, y)
5. 잠시 대기 후 다시 스크린샷 → 다음 단계 판단
```

**장점:**
- 캘리브레이션 불필요 — 매번 화면을 새로 인식
- 앱 업데이트/레이아웃 변경에도 동작
- 사이즈 개수, 스크롤 필요 여부, 품절 여부를 AI가 알아서 판단
- 상점명의 신뢰 표시(品牌官方, 得物 등)도 AI가 읽음

**구현 코드 구조:**

```python
import pyautogui
import anthropic
import base64
import time

client = anthropic.Anthropic()  # ANTHROPIC_API_KEY 환경변수에서 자동 로드

def capture_screen():
    """화면 캡처 → base64 인코딩"""
    screenshot = pyautogui.screenshot()
    # PIL Image → base64
    import io
    buffer = io.BytesIO()
    screenshot.save(buffer, format='PNG')
    return base64.b64encode(buffer.getvalue()).decode()

def ask_claude_vision(image_base64, prompt):
    """Claude API에 스크린샷 + 질문 전송"""
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/png",
                        "data": image_base64
                    }
                },
                {
                    "type": "text",
                    "text": prompt
                }
            ]
        }]
    )
    return response.content[0].text

def find_and_click(description):
    """화면에서 요소를 찾아 클릭"""
    img = capture_screen()
    prompt = f"""이 识货 앱 스크린샷에서 '{description}'의 화면 좌표(x, y)를 JSON으로 알려줘.
    응답 형식: {{"x": 숫자, "y": 숫자}}
    요소가 없으면: {{"x": null, "y": null, "reason": "이유"}}"""
    result = ask_claude_vision(img, prompt)
    coords = json.loads(result)
    if coords["x"] is not None:
        pyautogui.click(coords["x"], coords["y"])
    return coords

def read_screen_data(prompt):
    """화면에서 데이터 읽기 (클릭 없이)"""
    img = capture_screen()
    result = ask_claude_vision(img, prompt)
    return result
```

**사이즈 수집 예시:**
```python
# 사이즈 목록을 AI가 알아서 파악
sizes_info = read_screen_data("""
이 识货 앱 화면에서 사이즈 바를 분석해줘.
1. 보이는 모든 사이즈와 각 가격을 JSON 배열로 반환
2. 缺货(품절)인 사이즈는 price를 null로
3. 오른쪽에 스크롤하면 더 있는지 여부
응답 형식:
{
  "sizes": [{"size": "38", "price": 314}, {"size": "39.5", "price": null}],
  "has_more_right": true
}
""")
```

**판매처 신뢰도 판별 예시:**
```python
sellers_info = read_screen_data("""
이 识货 앱의 전체최저가(查看全网底价) 화면에서 판매처 리스트를 분석해줘.
각 판매처의 상점명, 플랫폼, 가격, 신뢰 표시를 추출해.

신뢰 가능 기준:
- 得物 (得物-品牌官方 포함) → trust: true
- 상점명 옆에 品牌官方 표시 → trust: true
- 상점명에 官方旗舰店 또는 旗舰店 포함 → trust: true
- 상점명에 专营店 포함 → trust: true
- 그 외 → trust: false

응답 형식:
{
  "sellers": [
    {"shop": "得物", "platform": "得物", "price": 314, "trust": true, "trust_reason": "得物"},
    {"shop": "斯谱运动", "platform": "识货", "price": 326, "trust": false, "trust_reason": "일반 상점"}
  ]
}
""")
```

## 알려진 제한사항 & 주의

- **识货 DB 정확도:** 100%가 아님. 연동 오류, 미입점 업체 있음. 품번 교차 확인 필수
- **가격 실시간 변동:** 수집 시점을 반드시 기록
- **품절 사이즈:** 缺货 표시 → null로 기록, 가격 수집 스킵
- **사이즈 바:** AI가 보이는 사이즈 개수를 판단하고, 더 있으면 스크롤 → 없으면 바로 수집
- **API 응답 파싱:** Claude가 JSON이 아닌 텍스트를 반환할 수 있으므로 파싱 에러 핸들링 필수
- **화면 전환 대기:** 클릭 후 화면이 바뀌는 데 시간이 걸림 → `time.sleep(1~2)` 후 스크린샷
