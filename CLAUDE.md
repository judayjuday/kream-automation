# KREAM Automation - Claude Code 작업 규칙

## 기본 규칙
1. 한국어로 작업
2. 작업 완료 후 반드시 구문 확인 (python3 -c "import py_compile; py_compile.compile('파일명', doraise=True)")
3. 구문 확인 통과하면 서버 재시작하고 테스트
4. 테스트 실패하면 스스로 디버깅하고 재시도 (최대 3회)
5. 각 작업 완료 후 결과 요약 출력하고 다음 작업으로 넘어가기
6. 해결 안 되는 문제는 해당 작업에서 멈추고 상세 보고
7. **데이터 수집 실패 시 절대 다른 데이터로 대체하지 마라. 실패는 실패로 표시하고, 사용자가 판단하도록 해라.**
   - 즉시구매가 수집 실패 → "수집 실패" 표시, 입찰 예정가 비움, 자동 입찰에서 제외
   - 전체 상품 최저가를 사이즈별 즉시구매가로 사용 금지
   - API 캡처 실패 → 경고 배너 표시, 세션 확인 유도
   - 마진 계산 불가 → "계산 불가" 표시, 체크박스 해제

## 서버 관련
- 서버 재시작: lsof -ti:5001 | xargs kill -9 2>/dev/null; python3 kream_server.py > server.log 2>&1 &
- auth_state.json 절대 빈 세션으로 덮어쓰지 말 것
- 성공 시에만 세션 저장

## 파일 구조
- kream_server.py: Flask 백엔드 (포트 5001)
- kream_bot.py: Playwright 자동화 (로그인/고시정보/입찰)
- kream_dashboard.html: 웹 대시보드 프론트엔드
- kream_collector.py: KREAM 가격 수집
- kream_adjuster.py: 가격 자동 조정
- competitor_analysis.py: 경쟁사 분석
- tabs/: 탭별 HTML 파일
- price_history.db: SQLite DB (가격이력/입찰이력/판매이력)

## 순차 작업 큐 (위에서부터 순서대로 진행)

### 작업 1: KREAM 세션 만료 시 자동 재로그인 검증 및 보강
- ensure_logged_in()에서 _try_auto_relogin() 호출이 실제로 동작하는지 테스트
- 세션 만료 상태를 시뮬레이션해서 자동 재로그인이 되는지 확인
- 실패 시 대시보드에 "세션 만료 - 재로그인 필요" 알림
- /api/session/status 엔드포인트에서 세션 유효성 확인 결과 정확히 반환

### 작업 2: 서버 크래시 시 자동 재시작 (watchdog)
- 서버가 죽으면 자동으로 다시 시작하는 스크립트 생성
- start_server.sh 파일 생성: 무한 루프로 서버 감시 + 재시작
- 크래시 시 로그 기록 (crash_log.txt)
- 사용법: bash start_server.sh (백그라운드 실행)

### 작업 3: 큐에 상품 추가할 때 KREAM 존재 여부 즉시 확인
- 모델번호 입력 후 "+큐에 추가" 클릭 시 KREAM에 해당 상품이 있는지 즉시 확인
- 없으면 "KREAM에 등록되지 않은 모델번호입니다" 경고 후 추가 차단
- 있으면 상품명을 미리 표시하고 큐에 추가

### 작업 4: 일괄 실행 진행률 표시
- 5개 상품 실행 중이면 "3/5 완료 (60%)" 프로그레스바 표시
- 현재 처리 중인 상품 모델번호도 표시
- 실시간 업데이트 (기존 폴링 활용)

### 작업 5: 입찰 결과 요약 카드
- 일괄 실행 또는 자동 입찰 완료 후 요약 표시
- "성공 N건, 실패 M건, 마진부족 K건, 수집실패 L건"
- 총 예상 수익, 평균 마진율도 표시
- 결과 로그 영역 상단에 요약 카드로 표시

### 작업 6: 자동 입찰 후 실제 반영 확인
- 자동 입찰 완료 후 /api/my-bids 자동 동기화
- 입찰한 상품이 실제 판매자센터에 등록됐는지 확인
- 결과 테이블의 "내 입찰가" 칼럼 자동 업데이트
- 등록 안 된 건은 "등록 확인 실패" 경고

### 작업 7: 환율 변동 알림
- 환율이 전일 대비 2% 이상 변동되면 대시보드에 알림 배너
- "⚠ CNY 환율 변동: 215.8 → 220.3 (+2.1%)" 형태
- 기존 큐의 마진이 달라질 수 있으니 재계산 권장 알림

### 작업 8: 가격 수집 시 배송타입별 가격 분리 저장
- price_history DB에 배송타입 필드가 이미 있으니 활용
- 빠른배송/일반배송/해외배송 각각의 가격을 별도 레코드로 저장
- 대시보드 상세 패널에서 배송타입별 가격 비교 테이블 표시
