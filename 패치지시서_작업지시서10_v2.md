## 수정 사항

### 수정 1 (★ 치명): imported_at 컬럼명 검증 후 일괄 치환
- 작업지시서_10의 다음 위치에 사용된 `imported_at`을 사전-1 결과로 검증
  - Step 2-1 _save_bid_cost 내부 `ORDER BY imported_at DESC LIMIT 1`
  - Step 4-1 api_bid_cost_shihuo_diff 내부 `ORDER BY imported_at DESC LIMIT 1`
  - 부록 컬럼표
- 만약 실제 컬럼이 다른 이름이면 모두 치환. 컬럼이 아예 없으면 ORDER BY 절을 `id DESC`로 변경 (모든 테이블에 id PK 존재).

### 수정 2 (★ 치명): Step 6 회귀 테스트의 Python REPL 방식 변경
기존:
```python
from kream_server import _save_bid_cost
r = _save_bid_cost(...)
```
문제: kream_server.py를 import하면 Flask 앱 초기화 부작용 + 포트 충돌 위험.

변경: Python REPL에서 _save_bid_cost를 직접 호출하지 않고, sqlite3로 직접 INSERT/SELECT 또는 임시 테스트 라우트 사용.

권장 방식: sqlite3 직접 INSERT로 시뮬레이션
```bash
# 시나리오 A: 식货 자동 채택 시뮬레이션을 SQL로
# 1) 식货에서 매칭 가능한 (model, kream_mm) 찾기
sqlite3 -header price_history.db "SELECT model, kream_mm, cny_price FROM shihuo_prices WHERE active=1 AND kream_mm IS NOT NULL LIMIT 1"

# 2) _save_bid_cost를 호출하는 대신, 같은 로직을 SQL로 검증
# (실제 함수 동작은 Step 6 별도 시나리오에서 자동 입찰 테스트로 검증)

# 또는 임시 테스트 엔드포인트를 작업지시서에 추가:
# /api/_test/save-bid-cost (DEBUG=True일 때만 등록)
```
작업지시서 v2에서는 Step 6의 검증 방식을 다음 중 택일하여 명시:
A안: sqlite3 직접 SQL로 동작 시뮬레이션
B안: 임시 테스트 엔드포인트 신설 (DEBUG 가드)
권장: A안 (코드 변경 최소)

### 수정 3 (★ 치명): Step 2-2-d auto_rebid의 order_id 무한 누적 방지
기존:
```python
order_id=f"{product_id}_{size}_rebid_{int(time.time())}"
```
변경: 기존 패턴 재사용 또는 UPSERT 의도 보존
```python
order_id=f"{product_id}_{size}"
```
또는 자동 재입찰의 의도가 매 건마다 별도 기록이라면 별도 테이블 사용을 권장 표시.
v2에서 두 가지 옵션을 명시하고 사용자에게 결정 요청 표기.

### 수정 4 (★ 치명): Step 7-3 DROP COLUMN 안내 안전화
사전-2에서 확인한 SQLite 버전이 3.35 미만이면 Step 7-3 전체를 다음으로 교체:
```
SQLite 버전 < 3.35로 DROP COLUMN 미지원.
컬럼 롤백이 필요하면 다음만 수행:
  UPDATE bid_cost SET cny_source=NULL WHERE cny_source IS NOT NULL
컬럼 자체는 유지 (NULL 허용이라 무해).
```

### 수정 5 (보강): Step 2-1 락 충돌 완화
_save_bid_cost 내부의 두 번의 sqlite3.connect()를 단일 connection으로 통합.
변경 예시 — 함수 본문에서 connection을 하나만 만들고 shihuo 조회/UPSERT 모두 그 cursor로 수행.

### 수정 6 (보강): Step 5 UI 위치 확정
사전-3 grep 결과를 작업지시서 Step 5 본문에 명시:
"식货 임포트 UI는 [실제 파일명] 라인 [번호] 근처. 그 위치에 '차이 보기' 버튼/모달 삽입."

### 수정 7 (보강): Step 1 마이그레이션 트랜잭션 명시
기존:
```python
c.execute("ALTER TABLE ...")
c.execute("UPDATE ...")
conn.commit()
```
변경:
```python
c.execute("BEGIN")
c.execute("ALTER TABLE ...")
c.execute("UPDATE ...")
conn.commit()
```
