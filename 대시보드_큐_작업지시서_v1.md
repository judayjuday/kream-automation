## 사용자 플로우

### 1단계: 상품 추가 (반복)

사용자가 최소한의 정보만 입력하면 큐에 추가됨:

**필수 입력:**
- 모델번호/품번 (예: IX7694, ID6016, 1203A408-200)
- 중국 구매가 (CNY)

**선택 입력 (안 넣으면 자동 판별):**
- 카테고리 (가방/신발/의류/기타) → 자동 판별 로직 있음 (아래 참조)
- 사이즈/옵션 (안 넣으면 "전사이즈" 또는 KREAM 검색 후 자동)
- 해외배송비 (기본값 8,000원)
- 수량 (기본값 1)

**추가 버튼** 클릭 → 큐 테이블에 한 줄 추가 → 입력 필드 초기화 → 바로 다음 상품 입력 가능

### 2단계: 큐 확인/수정

큐 테이블에 추가된 상품들이 리스트로 보임:

| # | 모델번호 | 카테고리 | 중국가(CNY) | 배송비 | 사이즈 | 수량 | 상태 | 액션 |
|---|----------|----------|-------------|--------|--------|------|------|------|
| 1 | IX7694 | 가방 (자동) | 220 | 8,000 | ONE SIZE | 1 | 대기 | 수정/삭제 |
| 2 | ID6016 | 신발 (자동) | 314 | 8,000 | 전사이즈 | 1 | 대기 | 수정/삭제 |
| 3 | 1203A408-200 | 신발 (수동) | 280 | 8,000 | 전사이즈 | 1 | 대기 | 수정/삭제 |

- 각 행에서 **수정/삭제** 가능
- 카테고리가 자동 판별된 경우 "(자동)" 표시, 사용자가 변경 가능
- 전체 삭제 버튼

### 3단계: 일괄 실행

**"일괄 실행" 버튼** 클릭 시:

```
각 상품마다:
1. KREAM 검색 (모델번호로) → KREAM 상품번호, 영문명, 사이즈 옵션 가져오기
2. 카테고리 자동 판별 (아직 미설정인 경우)
3. 마진 계산 (중국가 + 환율 + 관세 + 배송비 → KREAM 수수료 차감)
4. 결과 업데이트
```

상태가 실시간으로 변함: 대기 → 검색 중 → 계산 중 → 완료/실패

### 4단계: 결과 확인 + 선택 등록

일괄 실행 완료 후 결과 테이블:

| # | 모델번호 | 상품명 | 카테고리 | 원가(KRW) | KREAM 최저가 | 마진(10%) | 마진(15%) | 마진(20%) | 선택 |
|---|----------|--------|----------|-----------|-------------|-----------|-----------|-----------|------|
| 1 | IX7694 | Adidas Bag... | 가방 | 63,000 | 89,000 | ✅ +12,000 | ✅ +7,500 | ❌ | ☑ |
| 2 | ID6016 | Adidas VL Court... | 신발 | 85,000 | 95,000 | ❌ -2,000 | ❌ | ❌ | ☐ |
| 3 | 1203A408-200 | Asics... | 신발 | 78,000 | 120,000 | ✅ +28,000 | ✅ +22,000 | ✅ +16,000 | ☑ |

- ✅ = 마진 남음, ❌ = 손해
- 마진 나는 상품만 체크 → **"선택 상품 등록/입찰"** 버튼

## 카테고리 자동 판별 로직

KREAM 검색 결과의 **영문명**으로 판별한다 (识货 데이터는 사용하지 않음):

```python
def detect_category(english_name):
    name = english_name.lower()
    
    # 가방 (관세 8%)
    bag_keywords = ['bag', 'backpack', 'tote', 'pouch', 'wallet', 'clutch', 
                    'purse', 'satchel', 'rucksack', 'crossbody', 'shoulder bag',
                    'duffle', 'messenger', 'fanny pack', 'waist bag']
    
    # 신발 (관세 13%)
    shoe_keywords = ['shoe', 'sneaker', 'boot', 'sandal', 'slipper', 
                     'runner', 'trainer', 'loafer', 'mule', 'clog',
                     'slide', 'flip flop', 'oxford', 'derby']
    
    # 의류 (관세 13%)
    clothing_keywords = ['jacket', 'hoodie', 'shirt', 'pants', 'shorts',
                        'dress', 'skirt', 'coat', 'sweater', 'cardigan',
                        'vest', 'tee', 't-shirt', 'jogger', 'track']
    
    for kw in bag_keywords:
        if kw in name:
            return {"category": "가방", "tariff": 8, "auto": True}
    
    for kw in shoe_keywords:
        if kw in name:
            return {"category": "신발", "tariff": 13, "auto": True}
    
    for kw in clothing_keywords:
        if kw in name:
            return {"category": "의류", "tariff": 13, "auto": True}
    
    return {"category": None, "tariff": None, "auto": False}
    # → 사용자에게 직접 선택하도록 UI에 표시
```

## 마진 계산 공식

인수인계서 v2 기준:

```python
def calculate_margin(cny_price, category_tariff, shipping_krw=8000):
    # 환율 환산
    exchange_rate = 218.12  # CNY → KRW (설정에서 가져옴)
    exchange_margin = 1.03
    krw_price = cny_price * exchange_rate * exchange_margin
    
    # 관세 (USD 150 초과 시)
    usd_rate = 1495.76
    usd_value = krw_price / usd_rate
    if usd_value > 150:
        tariff = krw_price * (category_tariff / 100)
    else:
        tariff = 0
    
    # 총 원가
    total_cost = krw_price + tariff + shipping_krw
    
    # KREAM 수수료 역산 (판매가에서 차감)
    # 판매수수료 = 판매가 × 6%
    # 부가세 = 수수료 × 10%
    # 고정수수료 = 2,500원
    # 정산금 = 판매가 - (판매가 × 6% × 1.1) - 2,500
    
    # 마진별 필요 판매가 계산
    margins = {}
    for target_margin_pct in [0, 10, 15, 20]:
        target_profit = total_cost * (target_margin_pct / 100)
        required_net = total_cost + target_profit
        # required_net = sell_price - (sell_price × 0.066) - 2500
        # required_net = sell_price × 0.934 - 2500
        # sell_price = (required_net + 2500) / 0.934
        sell_price = (required_net + 2500) / 0.934
        margins[f"margin_{target_margin_pct}"] = {
            "sell_price": round(sell_price),
            "profit": round(sell_price * 0.934 - 2500 - total_cost),
        }
    
    return {
        "krw_price": round(krw_price),
        "tariff": round(tariff),
        "shipping": shipping_krw,
        "total_cost": round(total_cost),
        "margins": margins
    }
```

## 기술 구현 참고

### 백엔드 (kream_server.py)

새 API 엔드포인트:
- `POST /api/queue/add` — 큐에 상품 추가
- `GET /api/queue/list` — 큐 목록 조회
- `DELETE /api/queue/{id}` — 큐에서 삭제
- `PUT /api/queue/{id}` — 큐 항목 수정
- `POST /api/queue/execute` — 일괄 실행 (KREAM 검색 + 마진 계산)
- `DELETE /api/queue/clear` — 큐 전체 삭제

큐 데이터는 메모리(리스트)에 저장해도 되고, 간단한 JSON 파일에 저장해도 됨.

### 프론트엔드 (kream_dashboard.html)

- 상품 등록/입찰 탭에 큐 UI 추가
- 상단: 입력 폼 (모델번호 + CNY 가격 + 선택 필드) + 추가 버튼
- 중간: 큐 테이블 (추가된 상품 리스트)
- 하단: 일괄 실행 버튼 + 결과 테이블

### 기존 코드 활용

- KREAM 검색은 이미 `kream_server.py`에 API가 있음 (kream_collector.py 연동)
- 환율/수수료 설정은 설정 탭에서 가져오면 됨
- 마진 계산기 기능도 이미 있으므로 로직 재활용 가능
