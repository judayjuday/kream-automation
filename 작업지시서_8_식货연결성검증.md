# 작업지시서 8 — 식货 임포트→매핑→자동입찰 연결성 검증

목적: ID6016 테스트 엑셀로 end-to-end 검증.
모드: 검증만, 코드 수정 금지, 실제 입찰 금지.

테스트 데이터:
- ID6016 / 运动鞋 / EU 35.5 / 215mm / CNY 299
- ID6016 / 运动鞋 / EU 36 / 220mm / CNY 369

## 절대 규칙
1. price_history.db DROP/DELETE 금지
2. 실제 입찰 실행 금지 (place_bid, modify_bid_price 호출 금지)
3. settings.json 변경 금지
4. 검증 끝나면 반드시 Step 9 롤백
5. 각 Step 결과를 짧게 요약하면서 진행

## 사전 백업
cp price_history.db price_history_backup_test_id6016.db

## Step 1: 사전 상태 스냅샷
- curl /api/shihuo/latest
- sqlite3로 ID6016 기존 데이터 0건 확인

## Step 2: 브랜드 정규화 사전 검증 (가장 중요)
- normalize_brand 함수 위치 grep으로 찾기 (def normalize_brand)
- normalize_brand("adidas", "topsneaker旗舰店") 호출 결과 확인
- size_charts에 등록된 ADIDAS 브랜드 표기와 일치하는지 검증
- 불일치 시 중단하고 보고

## Step 3: ADIDAS 사이즈표 사전 확인
- sqlite3 "SELECT brand, gender, category, size_eu, kream_mm FROM size_charts WHERE brand LIKE '%ADIDAS%' AND size_eu IN ('35.5', '36');"
- EU 35.5→215, EU 36→220 등록 확인

## Step 4: 임포트 실행
- curl -X POST http://localhost:5001/api/shihuo/import -F "file=@识货_테스트_ID6016.xlsx"
- 응답에서 batch_id 추출하여 변수에 저장 (예: TEST_BATCH=$(...))
- 기대: mapped=2, no_size=0, mapping_failed=0, unknown_brand=0

## Step 5: DB 직접 검증
- sqlite3 "SELECT batch_id, active, brand_normalized, model, size_eu, kream_mm, cny_price, mapping_status FROM shihuo_prices WHERE model='ID6016' ORDER BY kream_mm;"
- 215mm/220mm 두 행이 mapped 상태로 저장됐는지 확인
- 이전 활성 batch가 active=0으로 변경됐는지 확인

## Step 6: API 조회 검증
- /api/shihuo/latest, /api/shihuo/by-model/ID6016, /api/shihuo/unmapped

## Step 7: 자동 입찰 엔진 연결성 분석 (코드 분석만, 실행 X)
- grep -rn "shihuo_prices" --include="*.py" . | grep -v CREATE | grep -v INDEX
- auto_adjust 엔진과 auto_rebid 모듈에서 식货 가격 참조하는지 전수 확인
- 결과: 참조 함수/파일/라인 보고. 미참조 시 명시.

## Step 8: my_bids 매칭 시뮬레이션 (실행 X, 계산만)
- my_bids_local.json에 ID6016 있는지 확인
- 가상 시나리오로 마진 계산:
  - 환율, 해외배송, 수수료 settings.json에서 로드
  - CNY → 원가 = CNY × 환율 × 1.03 + 8000
  - 정산 = 판매가 × (1 - 0.06 × 1.1) - 2500
  - 마진 = 정산 - 원가
  - 215mm CNY 299 가상판매가 350,000원 / 220mm CNY 369 가상판매가 420,000원

## Step 9: 롤백 (필수)
- curl -X POST /api/shihuo/rollback/$TEST_BATCH
- sqlite3 "UPDATE shihuo_prices SET active=1 WHERE batch_id='shihuo_20260501_121000';"
- /api/shihuo/latest로 활성 batch 복구 확인
