## 1. 비전 (Why)

### 한 문장 비전
> "주데이가 직접 검토/수정/관리하는 시간을 0에 가깝게 줄이고, KREAM 월매출 5,000만원을 달성한다."

### 배경
- 주데이는 1인 사업자로 두 비즈니스 운영:
  - **KREAM**: 스니커즈 리셀 (중국 Dewu 소싱)
  - **SSRO**: 자사몰 + 에이블리/지그재그/크로켓/네이버 (의류, 가방, 잡화)
- 현재 가장 큰 병목: **주데이가 매번 직접 들어가서 검토하고, 수정하고, 관리해야 하는 일** (시간 절도)
- 자동화가 늘어날수록 사업이 커지지만, 자동화 시스템 자체가 또 다른 관리 부담이 되어선 안 됨

### 성공 정의 (6개월 후)
| 지표 | 현재 | 6개월 목표 |
|------|------|-----------|
| KREAM 월매출 | 진행 중 | **5,000만원** |
| 주데이 일일 수동 작업 시간 | 미측정 | **2시간 이내** |
| KREAM 자동 운영률 | ~70% | **95% 이상** |
| SSRO 자동 운영률 | ~0% | **70% 이상** |
| CS 답변 초안 자동 생성률 | 0% | **80% 이상** |

## 2. 핵심 원칙 (How — 절대 변하지 않는 룰)

### 원칙 1: 안전이 속도보다 우선
- 모든 자동화 기능은 **기본 OFF**, 주데이 명시적 승인 후 ON
- 모든 위험 작업(삭제, 수정)은 **유예 시간 + 되돌리기 가능** 구조
- 판매 완료 건은 **절대 수정/삭제 금지**
- 원가 없으면 **NULL**, 가짜 값 절대 금지

### 원칙 2: 직접 작업 시간을 0으로
- 주데이가 "직접 들어가서 봐야 하는 일"을 발견하면 → 다음 작업 우선순위 1순위
- 알림/이메일은 "정보 제공"용이지 "수동 작업 트리거"가 되면 안 됨
- 대시보드는 **모니터링용**이지 **데이터 입력용**이 아님

### 원칙 3: 기능별 격리 (Sub-Agents 구조)
- 한 기능 수정 시 다른 기능에 영향 주면 안 됨
- 각 도메인(KREAM/SSRO/CS)은 독립적으로 배포/롤백 가능
- 새 플랫폼 추가가 기존 코드를 망가뜨리면 안 됨

### 원칙 4: 단일 진실 소스 (Single Source of Truth)
- 도메인별 `HANDOFF.md`가 그 도메인의 **유일한 진실**
- 인수인계서가 코드와 다르면 → **인수인계서를 즉시 업데이트**
- 채팅 시작 시 항상 NORTH_STAR.md + 도메인 HANDOFF부터 읽음

### 원칙 5: 수익 직결 우선
- 모든 작업은 "이게 매출/시간에 어떻게 기여하는지" 명시 가능해야 함
- 명시 못 하는 작업은 우선순위 뒤로
- 5,000만원 매출 목표에서 멀어지는 기능은 만들지 않음

### 원칙 6: 자체 검증 필수 (Verification Protocol)
- 모든 작업은 **Plan → Act → Verify → Report** 프로토콜 준수
- 완료 선언 전 자동 검증 4종 필수:
  1. 문법 체크 (py_compile)
  2. 서버 재시작 + 헬스체크 (curl /api/health)
  3. 관련 API 응답 확인
  4. 회귀 테스트 (pytest, 해당 도메인만)
- 검증 실패 시 → 완료 선언 금지, 주데이에게 보고
- 상세: VERIFICATION_PROTOCOL.md 참조

### 원칙 7: 모든 것을 기록하고 학습한다 (Observability + Learning Loop)
- 모든 실행/실패/경쟁자 행동 로그로 기록
- Cron 기반 자가 진단 (매일 새벽 3시 + 매주 일요일 밤)
  - 실패 패턴 자동 발견
  - 주데이에게 리포트 + 수정 제안
- 데이터 축적 → 시스템 자체 개선 (1년 후 훨씬 똑똑해짐)
- snapshot으로 되돌리기 가능 (시점 복원)
- 상세: OBSERVABILITY.md 참조

```

├── 💬 도메인 C: CS 자동화 [신규]
│   • 매일 50~100건 상담 답변 초안 자동 생성
│   • 출근 전 미리 준비 → 상담사 검토만
│
├── 📊 도메인 D: 통합 대시보드 + 데이터 [신규]
│   • 6개 도메인 통합 모니터링
│   • 매출/재고/판매 패턴 분석
│   • 알림 통합
│
├── 🎨 도메인 E: 이미지 자동 편집 (JUDAY) [신규]
│   • 디자이너 이승민님 업무 자동화
│   • 월 5개 신상품 → 월 15시간 절약 목표

│   • SSRO 상품마스터 자동 채움
│
└── 🔍 도메인 F: 신상품 정보 수집 [신규]
    • 샤오홍슈 + 다른 사이트 크롤링
    • 상품 정보 자동 정리
    • → 도메인 E 이미지 편집 트리거
    • → 도메인 B SSRO 상품 등록 연계
```

각 도메인은 **독립적으로 개발/배포/롤백 가능.**
