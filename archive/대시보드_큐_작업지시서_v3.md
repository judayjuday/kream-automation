## 1. 고시정보 자동 입력

### 현재 고시정보 필드 (KREAM 판매자센터 기준)

상품마다 다른 것만 입력:
- **종류** — 숄더백, 토트백, 크로스백... (가방) / 러닝화, 스니커즈... (신발)
- **소재** — 데님, 가죽, 나일론... / 합성피혁, 메쉬...
- **색상** — 블루, 블랙...
- **크기** — 50*20*30cm 또는 상품별 상이 / 260mm 등
- **제조자/수입자** — Adidas, Nike...

나머지는 설정 기본값 자동 적용 (이미 구현됨):
- 제조국: "상품별 상이 (케어라벨 참고)"
- 주의사항: 기본 문구
- 품질보증: "관련 법 및 소비자 분쟁 해결 기준에 따름"
- AS전화번호: "01075446127"
- 원산지: "China (중국) (CN)"
- HS코드: 가방 "4202.92" / 신발 "6404.11"

### 자동 채움 로직

KREAM 검색 결과의 **상품명(영문/한글)에서 직접 추출**한다.

예시: IX7694 검색 결과
- 영문: "Adidas Always Original Denim Shoulder Bag Black"
- 한글: "아디다스 올웨이즈 오리지널 데님 숄더백 블랙"

여기서 추출:
- 제조자/수입자: **Adidas** (브랜드 필드)
- 종류: **숄더백** (Shoulder Bag → 숄더백)
- 소재: **데님** (Denim → 데님)
- 색상: **블랙** (Black → 블랙)

```python
def auto_fill_from_product_name(kream_data):
    """KREAM 상품명에서 고시정보 자동 추출"""
    
    eng_name = kream_data.get("english_name", "").lower()
    kor_name = kream_data.get("korean_name", "")
    brand = kream_data.get("brand", "")
    
    info = {}
    
    # 1. 제조자/수입자 — 브랜드 필드에서
    info["manufacturer"] = brand  # "Adidas", "Nike" 등
    
    # 2. 종류 — 상품명에서 상품 유형 추출
    type_map = {
        # 가방
        "shoulder bag": "숄더백", "tote bag": "토트백", "tote": "토트백",
        "crossbody": "크로스백", "backpack": "백팩", "rucksack": "백팩",
        "pouch": "파우치", "wallet": "지갑", "clutch": "클러치",
        "duffle": "더플백", "messenger": "메신저백", "waist bag": "웨이스트백",
        "fanny pack": "웨이스트백", "bucket bag": "버킷백", "hobo": "호보백",
        # 신발
        "running shoe": "러닝화", "sneaker": "스니커즈", "slide": "슬라이드",
        "sandal": "샌들", "boot": "부츠", "loafer": "로퍼",
        "trainer": "트레이너", "slipper": "슬리퍼",
        # 의류
        "hoodie": "후드", "jacket": "자켓", "t-shirt": "티셔츠",
        "pants": "팬츠", "shorts": "쇼츠", "sweater": "스웨터",
    }
    for eng, kor in type_map.items():
        if eng in eng_name:
            info["type"] = kor
            break
    else:
        # 영문에서 못 찾으면 한글명에서 시도
        kor_types = ["숄더백", "토트백", "크로스백", "백팩", "파우치", "지갑",
                     "러닝화", "스니커즈", "슬라이드", "샌들", "부츠",
                     "후드", "자켓", "티셔츠", "팬츠"]
        for kt in kor_types:
            if kt in kor_name:
                info["type"] = kt
                break
        else:
            info["type"] = "상품별 상이"
    
    # 3. 소재 — 상품명에서 소재 키워드 추출
    material_map = {
        "denim": "데님", "leather": "가죽", "nylon": "나일론",
        "canvas": "캔버스", "suede": "스웨이드", "mesh": "메쉬",
        "cotton": "코튼", "polyester": "폴리에스터", "wool": "울",
        "silk": "실크", "fleece": "플리스", "gore-tex": "고어텍스",
        "rubber": "고무", "synthetic": "합성", "knit": "니트",
        "terry": "테리", "corduroy": "코듀로이", "satin": "새틴",
        "velvet": "벨벳", "linen": "리넨",
    }
    materials_found = []
    for eng, kor in material_map.items():
        if eng in eng_name:
            materials_found.append(kor)
    info["material"] = ", ".join(materials_found) if materials_found else "상품별 상이"
    
    # 4. 색상 — 상품명 마지막 단어(들)에서 추출
    color_map = {
        "black": "블랙", "white": "화이트", "red": "레드",
        "blue": "블루", "navy": "네이비", "green": "그린",
        "grey": "그레이", "gray": "그레이", "pink": "핑크",
        "beige": "베이지", "brown": "브라운", "cream": "크림",
        "orange": "오렌지", "yellow": "옐로", "purple": "퍼플",
        "silver": "실버", "gold": "골드", "olive": "올리브",
        "burgundy": "버건디", "khaki": "카키", "ivory": "아이보리",
        "coral": "코랄", "mint": "민트", "charcoal": "차콜",
    }
    colors_found = []
    for eng, kor in color_map.items():
        if eng in eng_name:
            colors_found.append(kor)
    info["color"] = ", ".join(colors_found) if colors_found else "상품별 상이"
    
    # 5. 크기 — 기본값
    info["size"] = "상품별 상이"
    
    # 6. HS코드 + 관세 — 카테고리에 따라
    category = detect_category(eng_name)  # v1의 카테고리 판별 함수 사용
    if category.get("category") == "가방":
        info["hs_code"] = "4202.92"
        info["tariff"] = 8
    elif category.get("category") == "신발":
        info["hs_code"] = "6404.11"
        info["tariff"] = 13
    else:
        info["hs_code"] = ""
        info["tariff"] = 13
    
    return info
```

### 고시정보 등록 여부 확인

- KREAM 판매자센터에서 해당 상품의 고시정보가 이미 등록되어 있는지 확인
- 이미 등록됨 → "이미 고시정보 등록됨 (입찰만 진행)" 체크박스 자동 체크
- 미등록 → 자동 채운 고시정보로 등록 진행

## 2. 일괄 실행 결과 테이블 (마진 확인 + 가격 선택 통합)

일괄 실행 후 하나의 큰 테이블에서 모든 것을 한눈에 처리:

### 테이블 구성

```
┌────┬──────────┬─────────────┬──────┬────────┬──────────┬─────────┬─────────┬─────────┬──────┬────────┬────┐
│ ☑  │ 모델번호 │   상품명    │카테고│원가(원)│KREAM최저가│마진10% │마진15% │마진20% │고시정보│ 상태  │액션│
├────┼──────────┼─────────────┼──────┼────────┼──────────┼─────────┼─────────┼─────────┼──────┼────────┼────┤
│ ☑  │ IX7694   │ Adidas Bag..│ 가방 │ 63,000 │  89,000  │+12,000 │ +7,500 │ +3,000 │ 미등록│  완료  │ 📝 │
│ ☐  │ ID6016   │ VL Court... │ 신발 │ 85,000 │  79,000  │ -8,500 │-13,500 │-18,500 │ 등록됨│  완료  │ 📝 │
│ ☑  │ 1203A408 │ Asics Gel...│ 신발 │ 78,000 │ 120,000  │+28,000 │+22,000 │+16,000 │ 미등록│  완료  │ 📝 │
└────┴──────────┴─────────────┴──────┴────────┴──────────┴─────────┴─────────┴─────────┴──────┴────────┴────┘
```

### 각 컬럼 설명

- **☑ (체크박스):** 등록/입찰할 상품 선택. 마진이 마이너스인 건 자동으로 체크 해제
- **모델번호:** 입력한 품번
- **상품명:** KREAM 검색으로 가져온 이름 (짧게 표시, 호버하면 풀네임)
- **카테고리:** 가방/신발/의류 — 클릭하면 드롭다운으로 변경 가능
- **원가(원):** CNY → KRW 환산 + 관세 + 배송비 합산
- **KREAM 최저가:** 현재 KREAM의 최저 판매 입찰가
- **마진 10%/15%/20%:** 각 마진율에서의 예상 수익(원). 양수면 초록, 음수면 빨강
- **고시정보:** "등록됨" / "미등록" 표시. 미등록이면 자동 채움된 정보로 등록 예정
- **상태:** 대기/검색 중/계산 중/완료/실패
- **액션 (📝):** 클릭하면 해당 상품의 상세 편집 모달 열림

### 상세 편집 모달 (📝 클릭 시)

각 상품을 개별 수정할 수 있는 모달:

```
┌─────────────────────────────────────────┐
│ IX7694 — Adidas Bag...           [닫기] │
├─────────────────────────────────────────┤
│                                         │
│ 기본 정보                               │
│ ├ 카테고리: [가방 ▼]                    │
│ ├ 중국 구매가: [220] CNY                │
│ ├ 해외배송비: [8000] 원                 │
│ ├ 사이즈: [ONE SIZE]                    │
│ └ 수량: [1]                             │
│                                         │
│ 고시정보                                │
│ ├ 종류: [상품별 상이]                   │
│ ├ 소재: [상품별 상이]                   │
│ ├ 색상: [블랙]       ← 자동 채움       │
│ ├ 크기: [상품별 상이]                   │
│ └ 제조자: [Adidas]   ← 자동 채움       │
│                                         │
│ 마진 상세                               │

│ ├ KREAM 최저가: 89,000원               │
│ ├ 손익분기 판매가: 70,000원            │
│ ├ 마진10% 판매가: 77,000원 (+7,000)    │
│ ├ 마진15% 판매가: 80,500원 (+10,500)   │
│ └ 마진20% 판매가: 84,000원 (+14,000)   │
│                                         │
│ 입찰 가격 선택                          │
│ ○ 손익분기  ○ 마진10%  ● 마진15%      │
│   → 입찰가: 80,500원                   │
│   → 예상 수익: +10,500원               │
│                                         │

└─────────────────────────────────────────┘
```

### 가격 선택 방식

두 가지 모드:

**모드 A — 일괄 기본 마진 설정 (빠른 처리)**
- 테이블 상단에 "기본 마진율: [15%▼]" 드롭다운
- 선택하면 전체 상품에 해당 마진율 적용
- 개별 수정 필요하면 📝 모달에서 변경

**모드 B — 개별 가격 선택 (정밀 처리)**
- 📝 모달에서 상품별로 마진율 또는 직접 가격 입력

기본은 모드 A로 빠르게 처리하고, 필요한 것만 모드 B로 세밀하게 조정.

## 3. 등록/입찰 실행

**"선택 상품 등록/입찰"** 버튼 클릭 시:

```
각 선택된 상품마다:
1. 고시정보 미등록 → 고시정보 자동 등록 (kream_bot.py의 product 모드)
2. 고시정보 등록됨 → 건너뜀
3. 선택된 가격으로 판매 입찰 등록 (kream_bot.py의 bid 모드)
4. 상태 업데이트: 등록 중 → 입찰 중 → 완료/실패
```

이 단계는 Playwright로 KREAM 판매자센터를 실제 조작하는 것이므로, 
대시보드에서 "실행" 버튼을 누르면 백엔드가 kream_bot.py를 호출한다.

## 5. 신발 사이즈별 가격 입력

### 입력 규칙

- **사이즈를 입력하지 않으면** → 신발이 아닌 상품으로 간주 (가방/의류 등). 사이즈 무시.
- **사이즈를 입력하면** → 신발. 사이즈별로 다른 가격을 넣을 수 있음.

### 사이즈 체계 선택

사용자가 **EU** 또는 **JP** 중 하나를 선택. 한 칸에만 넣어도 됨:

```
사이즈 체계: ○ EU (38, 38.5, 39...)  ● JP (240, 245, 250...)
```

- **EU:** 36, 36.5, 37, 37.5, 38, 38.5, 39, 39.5, 40, 40.5, 41, 42, 43, 44, 45, 46
- **JP:** 225, 230, 235, 240, 245, 250, 255, 260, 265, 270, 275, 280, 285, 290

### 큐 추가 시 입력 UI

가방/의류처럼 사이즈 없는 상품:
```
모델번호: [IX7694]  중국가(CNY): [220]  [+ 추가]
```

신발 (사이즈별 가격):
```
모델번호: [ID6016]  사이즈 체계: [EU ▼]

┌─────────┬───────────┬────────┐
│ 사이즈  │ 중국가CNY │  삭제  │
├─────────┼───────────┼────────┤
│  38     │   314     │   🗑   │
│  38.5   │   386     │   🗑   │
│  39     │   293     │   🗑   │
│  40     │   247     │   🗑   │
│  41     │   239     │   🗑   │
│  42     │   285     │   🗑   │
├─────────┼───────────┼────────┤
│ [     ] │ [       ] │ [+행]  │
└─────────┴───────────┴────────┘

[+ 큐에 추가]
```

- 사이즈 입력: EU면 38, 38.5 형식 / JP면 240, 245 형식
- 사이즈마다 중국가가 다를 수 있음 (识货에서 사이즈별 가격이 다르니까)
- 모든 사이즈가 같은 가격이면 한 행만 넣고 "전사이즈 동일가" 체크박스
- [+행] 버튼으로 사이즈 행 추가, 🗑로 삭제

### 전사이즈 동일가 모드

```
☑ 전사이즈 동일가
중국가(CNY): [314]
사이즈 범위: [38] ~ [42] (EU)
→ 38, 38.5, 39, 39.5, 40, 40.5, 41, 42 전부 314 CNY로 등록
```

### 큐 테이블에서 신발 표시

| # | 모델번호 | 카테고리 | 사이즈 | 중국가(CNY) | 배송비 | 상태 |
|---|----------|----------|--------|-------------|--------|------|
| 1 | IX7694 | 가방 | ONE SIZE | 220 | 8,000 | 대기 |
| 2 | ID6016 | 신발 | 38~42 (6사이즈) | 239~386 | 8,000 | 대기 |
| 3 | 1203A408 | 신발 | 전사이즈 동일 280 | 280 | 8,000 | 대기 |

신발은 사이즈 수와 가격 범위를 요약해서 보여줌. 📝 클릭하면 상세 사이즈별 가격 편집 가능.

### 결과 테이블에서 신발

일괄 실행 후 신발은 **사이즈별로 마진이 다르게** 계산됨:

```
ID6016 — Adidas VL Court 2.0 (신발)
├ 38  : 원가 85,000 / KREAM최저 95,000 / 마진15% +7,000 ✅
├ 38.5: 원가 100,000 / KREAM최저 89,000 / 마진15% -18,000 ❌
├ 39  : 원가 80,000 / KREAM최저 92,000 / 마진15% +4,000 ✅
├ 40  : 원가 70,000 / KREAM최저 88,000 / 마진15% +10,000 ✅
├ 41  : 원가 68,000 / KREAM최저 85,000 / 마진15% +9,000 ✅
└ 42  : 원가 77,000 / KREAM최저 90,000 / 마진15% +5,000 ✅
→ 마진 나는 사이즈만 자동 체크: 38, 39, 40, 41, 42 (5개)
→ 38.5는 자동 체크 해제 (손해)
```

사용자는 사이즈별로 개별 체크/체크해제 가능.
